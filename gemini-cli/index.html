Google 的 Gemini CLI，定位也非常直接：把 Gemini 这个 agent 直接塞进你的终端里。

项目地址：google-gemini/gemini-cli

它和一般“命令行聊天工具”的区别在于，它不是只接收一段 prompt 然后吐一段文字，而是已经做成了完整 agent：会读文件、改文件、跑 shell、抓网页、接 MCP，还支持 GEMINI.md 分层记忆、custom commands、hooks、skills、sandbox。

截至 2026-04-04，我打开 GitHub 仓库页时看到：

  • 仓库 star 大约 100k；
  • 最新 stable release 是 v0.36.0，发布日期 2026-04-01；
  • README 对它的定义是：“An open-source AI agent that brings the power of Gemini directly into your terminal.”

命令 说明
/auth 切换认证方式
/chat 保存、恢复、分享会话检查点
/clear 清空界面
/commands 重新加载自定义命令
/compress 压缩上下文，省 token
/directory 管理多目录工作区
/extensions 查看当前启用的扩展
/hooks 查看和启停 hooks
/init 分析当前目录并生成 GEMINI.md

/model 切换 Gemini 模型

/resume 恢复历史会话

除了 slash commands，它还支持：

  • @路径：把文件或目录内容注入提示词；
  • 自定义 commands：从 .toml 里加载快捷命令。

所以它的可编排程度已经相当高了。

GEMINI.md：它的“项目记忆”

官方文档：CLI Configuration

Gemini CLI 很有特点的一点，是它把 GEMINI.md 做成了 hierarchical instructional context。

文档里写得很明确：默认上下文文件名是 GEMINI.md，但也可以改成别的，比如 AGENTS.md。

它的层级加载顺序大概是：

  1. ~/.gemini/GEMINI.md 这样的全局上下文；
  2. 当前项目根目录以及父目录上的 context file；
  3. 当前目录下面子目录里的 context file。

而且 /memory show 可以直接把最终拼接出来的上下文看出来。

这套设计很实用，因为它允许你：

  • 全局写通用习惯；
  • 项目根目录写项目约定；
  • 某个子目录再写更细的局部规则。

这比“所有提示词都塞一个文件里”高级不少。

为什么 Gemini CLI 值得看

我觉得它最有意思的地方不是“Google 也出了个 CLI”，而是它已经把很多 agent runtime 的关键零件都做齐了：

  • 命令行交互和非交互模式；
  • GEMINI.md 分层记忆；
  • MCP；
  • hooks；
  • skills；
  • extensions；
  • sandbox；
  • 会话恢复；
  • 自定义 commands。

也就是说，它不是一个轻量 demo，而是明显朝着“可以长期挂在开发者终端里的 agent 平台”去做的。
